# AI Context: Plutus (YourFinance) Project

This document provides a comprehensive overview of the **Plutus** project to help AI systems and developers understand the architecture, data flow, and core logic.

## 1. PROJECT OVERVIEW
- **Project Name**: Plutus (YourFinance)
- **Main Purpose**: A comprehensive wealth management dashboard for tracking stocks, precious metals, and personal portfolios.
- **Problem it Solves**: Simplifies financial data analysis by providing real-time market data, AI-driven portfolio insights, and activity monitoring in one platform.
- **Key Features**:
  - Real-time Gold & Silver spot price tracking with 15-day predictions.
  - Sector-based stock browsing (IT, Banking, Automobile, etc.).
  - Interactive portfolio management with buy/sell simulation.
  - **AI Clustering**: K-Means clustering to group portfolio stocks by performance and volatility.
  - **Technical Analytics**: RSI (Relative Strength Index), MA20 (Moving Average), and P/E Ratio trends.
  - **Newsletter Service**: Subscription model with integrated dummy payment gateway.
  - **Activity Monitor**: Backend dashboard for tracking user signups, logins, and API usage.
  - **Plutus Assistant**: Intelligent chatbot for financial and app-related queries.
- **Target Users**: Retail investors, financial enthusiasts, and developers exploring fintech integrations.

## 2. SYSTEM ARCHITECTURE
- **Frontend**: React 19, Styled Components (CSS-in-JS), Framer Motion (Animations), Recharts (Data Visualization).
- **Backend**: Python 3.11, Django 4.2, Django REST Framework (API).
- **Database**: SQLite (Development), extensible to PostgreSQL/MySQL.
- **External APIs**: Yahoo Finance (`yfinance`), MetalPriceAPI (Gold/Silver spot rates).
- **Process Management**: PM2 (Production process monitoring and persistence).

## 3. PROJECT STRUCTURE
### Root
- `ecosystem.config.js`: PM2 configuration for backend and frontend services.
- `deploy.sh` / `deploy.bat`: Deployment automation scripts for Linux/Windows.
- `logs/`: Directory for service logs.

### Frontend (`/frontend`)
- `src/pages/`: Main views (Home, Stocks, Portfolio, GoldSilver).
- `src/components/`: Reusable UI components (Chatbot, Prediction charts, Analytics).
- `src/services/`: API communication logic (Auth, Stocks, Metals).
- `src/contexts/`: Global state management (AuthContext for user sessions and portfolio).
- `src/styles/`: Theme definitions and global CSS.

### Backend (`/backend`)
- `users/models.py`: Database schema (User, Stock, Portfolio, Activity, Payment).
- `users/views.py`: Business logic and API endpoints.
- `users/templates/`: Backend dashboards (Activity Monitor).
- `backend/settings.py`: Global Django configuration.

## 4. DATA FLOW
1. **Authentication**: User logs in → JWT tokens generated by Backend → Stored in Frontend `localStorage`.
2. **Market Data**: Frontend requests sector data → Backend fetches from `yfinance` cache or API → Returns JSON to Frontend.
3. **Portfolio**: User adds stock → Frontend sends request with JWT → Backend updates `PortfolioStock` and logs `UserActivity`.
4. **Activity Monitoring**: Every sensitive action (Login, Add Stock, Payment) triggers a `UserActivity` log entry viewable in the Backend dashboard.

## 5. KEY MODULES
- **Auth Module**: Handles registration, JWT-based login, and logout tracking.
- **Analytics Module**: Calculates RSI, MA20, and Correlation coefficients for market data.
- **AI Module**: Implements K-Means clustering on the frontend to categorize stocks.
- **Payment Module**: Simulates a secure subscription flow with dummy card processing.

## 6. IMPORTANT LOGIC
- **Linear Regression**: Used in `MetalsPredictionChart` to forecast 15-day price trends.
- **K-Means Clustering**: Groups stocks into "Growth", "Stable", and "Declining" clusters based on price change percentage.
- **Indicator Calculation**: `calculateIndicators` utility creates deep copies of data to inject technical signals (RSI/MA20) without mutating original state.

## 7. CONFIGURATION
- **Backend**: Managed in `settings.py`. Key settings include `ALLOWED_HOSTS`, `CORS_ALLOWED_ORIGINS`, and `DATABASES`.
- **Frontend**: Environment variables can be added to `.env` for `API_BASE` URLs.

## 8. INSTALLATION & SETUP
1. **Backend**:
   ```bash
   cd backend
   pip install -r requirements.txt
   python manage.py migrate
   python manage.py runserver
   ```
2. **Frontend**:
   ```bash
   cd frontend
   npm install
   npm start
   ```

## 9. DEPLOYMENT
- Use `pm2 start ecosystem.config.js` to run both services.
- Frontend is served on port `3000`, Backend API on port `8000`.

## 10. AI UNDERSTANDING SUMMARY
To extend this project, an AI agent should focus on adding new analytical components in `frontend/src/components` or new data processing logic in `backend/users/views.py`. Ensure all new user actions are logged using the `UserActivity` model to maintain dashboard consistency.
